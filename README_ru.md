# Koe

*[English](README.md) · **Русский***

Discord-бот для приватных голосовых каналов (join-to-create), написанный на TypeScript на базе NestJS + Necord.

---

## Возможности

- **Join-to-create** каналы: при заходе в обозначенный голосовой канал автоматически создаётся приватный канал и пользователь переносится в него.
- **Интерактивная панель управления** на кнопках для каждого сервера — какие кнопки показывать, их порядок и количество в ряду настраиваются через текстовую команду `!privates`.
- Действия владельца: выдача/забирание доступа, скрытие/показ, закрытие/открытие, мут/размут, переименование, кик, передача владения, лимит пользователей.
- **Защита владения** — правки прав на приватном канале, сделанные не владельцем, откатываются по аудит-логу.
- Пустые приватные каналы автоудаляются; состояние восстанавливается после рестарта.
- Allowlist администраторов на уровне сервера (user ID и role ID) + глобальный список владельцев бота, который обходит все ограничения.

## Конфигурация

Переменные окружения (`.env`):

| Переменная      | Назначение                      |
|-----------------|---------------------------------|
| `DISCORD_TOKEN` | токен бота                      |
| `DATABASE_URL`  | строка подключения к PostgreSQL |

Рантайм-конфигурация разбита на три файла в `src/config/`:

| Файл                 | Назначение                                                                       |
|----------------------|----------------------------------------------------------------------------------|
| `main.config.ts`     | Глобальные владельцы бота (обходят проверки владения везде) + цвет embed-ов      |
| `privates.config.ts` | Настройки по серверам: admin user ID, admin role ID, ID join-to-create каналов   |
| `emoji.config.ts`    | Эмодзи для embed-ов (разделители, пагинация)                                     |

Примеры всех трёх файлов лежат в `src/config/examples/` — скопировать в `src/config/` и заполнить своими значениями.

## Запуск на Linux/Windows

### 1. Клонирование

```bash
git clone https://github.com/yokkkoso/koe koe
cd koe
```

### 2. Окружение

```bash
cp .env.example .env
```

Заполнить `DISCORD_TOKEN` и `DATABASE_URL`.

### 3. Конфигурация

```bash
cp src/config/examples/main.config.ts     src/config/main.config.ts
cp src/config/examples/privates.config.ts src/config/privates.config.ts
cp src/config/examples/emoji.config.ts    src/config/emoji.config.ts
```

Заполнить своими значениями.

### 4. Зависимости и БД

```bash
corepack enable
yarn install
yarn prisma migrate deploy
```

### 5. Сборка и запуск

```bash
yarn build
yarn start:prod
```

Для разработки — `yarn start:dev` (hot reload).

## Использование

На настроенном сервере выполнить `!privates` под аккаунтом с правами администратора — бот ответит сообщением настройки панели. Добавить/удалить кнопки, задать порядок, выбрать количество в ряду, затем отправить итоговую панель в нужный канал. Конечные пользователи нажимают кнопки на этой панели для управления своим приватным голосовым каналом.
